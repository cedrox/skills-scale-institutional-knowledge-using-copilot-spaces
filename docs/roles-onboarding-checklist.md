# Roles Onboarding Checklist

Purpose: Ensure new team members and role owners are aligned on responsibilities, contacts, access, and ceremonies before contributing to the project.

See [docs/octoacme-roles-and-personas.md](octoacme-roles-and-personas.md) for full role definitions.

---

## Role Assignment & Documentation
- [ ] Confirm role assignment and add role owner name to project README
- [ ] Confirm role summary is up to date in [docs/octoacme-roles-and-personas.md](octoacme-roles-and-personas.md)

## Primary Contacts
- [ ] PM contact (name, GitHub, email/Slack) shared with new role owner
- [ ] PdM contact shared
- [ ] Scrum Master contact shared
- [ ] Technical Lead contact shared
- [ ] Release Engineer contact shared
- [ ] DevOps Engineer contact shared
- [ ] QA lead contact shared
- [ ] Customer Support Lead contact shared

## Access & Environment Setup
- [ ] Repository read/write access granted
- [ ] CI/CD and pipeline access granted (where applicable)
- [ ] Monitoring and alerting dashboards access granted (DevOps/Release Engineer/Support)
- [ ] Project board (e.g., GitHub Projects) access confirmed
- [ ] Deployment environment access granted (where applicable)
- [ ] On-call rotation added (DevOps/Release Engineer/Support)

## Orientation
- [ ] Project one-pager and current roadmap shared
- [ ] Existing documentation walkthrough completed (docs/ folder)
- [ ] Runbooks and monitoring dashboards location shared

## Initial Meetings
- [ ] Intro meeting with PM and PdM scheduled
- [ ] Team kickoff / meet-and-greet completed
- [ ] Technical walkthrough with Technical Lead (for engineering roles)
- [ ] Handoff meeting with Release Engineer / DevOps for deployment responsibilities (for release/ops roles)
- [ ] Support readiness walkthrough with Customer Support Lead (for support roles)

## Ceremonies
- [ ] Added to recurring standup
- [ ] Added to sprint planning
- [ ] Added to sprint demo / review
- [ ] Added to retrospective

## Ownership
- [ ] Added to Risk Register as owner for relevant risks
- [ ] Open action items reviewed and role owner assigned where applicable

---

> For release-specific readiness, see [docs/release-checklist.md](release-checklist.md).
